# Plano: Painel Secreto de Analytics Admin

## Objetivo

Criar uma página secreta de analytics acessível apenas por super admin em `/analytics`, com proteção de autenticação existente, banco de dados Appwrite para persistência, e remover o widget flutuante "A" do site.

---

## Arquitetura

```
[Cliente] → trackEvent/trackPageView → localStorage (atual) + Appwrite API (novo)
[Super Admin] → /analytics → SuperAdminRoute → AnalyticsAdminPage → Appwrite DB → Dashboard
```

### Dados rastreados
- **Page views**: URL, timestamp, dispositivo, viewport, user agent
- **Ações de usuários**: Cliques nomeados, formulários, scroll depth
- **Dados de sessão**: Session ID, duração, IP parcial, plataforma
- **Métricas de negócio**: Livros emprestados, usuários ativos, etc. (via polling de collections existentes)

## Fase 2 — Service de Analytics Appwrite

**Novo arquivo:** `src/services/analyticsService.js`

Funções:
- `trackAnalyticsEvent(event)` → escreve na collection `analytics_events` via Appwrite Databases
- `upsertAnalyticsSession(session)` → cria ou atualiza sessão na collection `analytics_sessions`
- `fetchAnalyticsEvents(filters)` → busca eventos com filtros (para o dashboard admin)
- `fetchAnalyticsSummary(dateRange)` → agregação de métricas
- `fetchAnalyticsSessions(filters)` → busca sessões

Usa `getAppwriteDatabases()` e `DATABASE_ID` existentes.

## Fase 6 — Página AnalyticsAdmin.jsx

**Novo arquivo:** `src/pages/AnalyticsAdmin.jsx`

Layout: Usa `MainLayout` com título "Analytics".

### Seções do Dashboard:

**1. KPI Cards (grid 4 colunas)**
- Total de eventos (7d / 30d)
- Sessões únicas
- Page views totais
- Scroll depth médio

**2. Gráfico de line chart** (recharts)
- Page views por dia (últimos 30 dias)
- using `ResponsiveContainer` + `LineChart`

**3. Tabela de páginas mais visitadas**
- URL, views, usuários únicos, scroll médio
- `Table` do shadcn/ui com `overflow-x-auto`

**4. Timeline de eventos recentes**
- Lista dos últimos 50 eventos
- Tipo, página, timestamp

**5. Sessões ativas**
- Lista de sessões com: ID, duração, page views, plataforma

**6. Filtros**
- Date range picker (últimos 7d, 30d, 90d, personalizado)
- Filtro por tipo de evento
- Filtro por página

### Padrões responsivos (seguindo convenções do projeto):
- `space-y-4 sm:space-y-6` para espaçamento
- `grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-4` para KPI cards
- `grid grid-cols-1 xl:grid-cols-2` para charts
- `p-4 sm:p-6` para padding
- `flex flex-col sm:flex-row` para headers com filtros

### Dados:
- Busca dados via `fetchAnalyticsEvents()` e `fetchAnalyticsSummary()`
- usa `@tanstack/react-query` com `useQuery` para cache automático
- Polling a cada 30s para dados em tempo real

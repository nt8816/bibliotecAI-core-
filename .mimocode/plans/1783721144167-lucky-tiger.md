# Plano: Analytics da Landing Page com Supabase + Dashboard Admin

## Objetivo

Migrar o sistema de analytics do Appwrite para Supabase e criar o painel admin `/analytics` acessível apenas por super admin, com dashboard completo (KPIs, gráficos, tabelas, timeline).

---

## Arquitetura

```
LandingPage → useAnalytics hooks → analyticsService.js → platformApi.js → Cloudflare Worker → Supabase REST
SuperAdmin → /analytics → SuperAdminRoute → AnalyticsAdmin.jsx → platformApi.js → Cloudflare Worker → Supabase REST
```

O frontend NÃO fala direto com Supabase para queries — tudo passa pelo API gateway (Cloudflare Workers) que usa service-role key.

---

## Fase 2 — Rotas no API Gateway (Cloudflare Worker)

**Arquivo:** `cloudflare/api-gateway/src/index.ts`

Adicionar 4 novas rotas no objeto `routes`:

### `POST /v1/analytics/events`
- Autenticação: NÃO requer (fire-and-forget do cliente anônimo)
- Body: `{ event_type, page_url, element_id, session_id, user_id?, data?, viewport_w, screen_w, platform, user_agent }`
- Action: INSERT INTO `analytics_events` via `supabaseAdminRequest`

### `POST /v1/analytics/sessions`
- Autenticação: NÃO requer
- Body: `{ session_id, user_id?, scrollDepth?, platform, userAgent }`
- Action: UPSERT em `analytics_sessions` (criar ou incrementar page_views e atualizar max_scroll_depth)

### `GET /v1/analytics/events`
- Autenticação: REQUER super admin (via `x-user-access-token`)
- Query params: `limit`, `event_type`, `page_url`, `start_date`, `end_date`
- Action: SELECT de `analytics_events` com filtros, ORDER BY created_at DESC

### `GET /v1/analytics/summary`
- Autenticação: REQUER super admin
- Query params: `days` (default 30)
- Action: Agregações — total events, sessions, page views by URL, event type counts, avg scroll depth

## Fase 3 — Refatorar analyticsService.js

**Arquivo:** `src/services/analyticsService.js`

Substituir completamente. Todas as chamadas Appwrite viram chamadas `requestPlatformApi`:

```js
import { requestPlatformApi } from '@/lib/platformApi';

export async function trackAnalyticsEvent(event) {
  // POST /v1/analytics/events (fire-and-forget, sem auth)
  return requestPlatformApi('/v1/analytics/events', {
    method: 'POST', body: event, auth: false
  });
}

export async function upsertAnalyticsSession(session) {
  // POST /v1/analytics/sessions (fire-and-forget)
  return requestPlatformApi('/v1/analytics/sessions', {
    method: 'POST', body: session, auth: false
  });
}

export async function fetchAnalyticsEvents({ limit, eventType, pageUrl, startDate, endDate }) {
  // GET /v1/analytics/events?... (com auth)
  const params = new URLSearchParams();
  if (limit) params.set('limit', limit);
  if (eventType) params.set('event_type', eventType);
  if (pageUrl) params.set('page_url', pageUrl);
  if (startDate) params.set('start_date', startDate);
  if (endDate) params.set('end_date', endDate);
  return requestPlatformApi(`/v1/analytics/events?${params}`, { auth: true });
}

export async function fetchAnalyticsSummary({ days = 30 } = {}) {
  // GET /v1/analytics/summary?days=30
  return requestPlatformApi(`/v1/analytics/summary?days=${days}`, { auth: true });
}
```

Remover: imports do Appwrite, `hashIp`, todas as funções antigas.

## Fase 6 — Página AnalyticsAdmin.jsx

**Novo arquivo:** `src/pages/AnalyticsAdmin.jsx`

### Layout
- Usa o padrão de páginas admin existentes (sem MainLayout — páginas admin como AdminTenants.jsx usam layout próprio com sidebar)
- Título: "Analytics"
- Filtros no topo (date range: 7d, 30d, 90d)
- Polling a cada 30s via `useQuery` do `@tanstack/react-query`

### Seções

**1. KPI Cards (grid 4 colunas responsivo)**
- Total de eventos
- Sessões únicas
- Page views totais
- Scroll depth médio

Componentes: `Card`, `CardHeader`, `CardContent`, `CardTitle` do shadcn/ui

**2. Gráfico de page views por dia** (LineChart do recharts)
- Wrapper: `ChartContainer` do `src/components/ui/chart.js`
- Dados: agrupados por data

**3. Tabela de páginas mais visitadas**
- Componente: `Table` do shadcn/ui (`src/components/ui/table.js`)
- Colunas: URL, Views, Eventos, Último acesso

**4. Timeline de eventos recentes**
- Lista dos últimos 50 eventos
- Badges coloridos por tipo de evento
- Timestamp relativo

**5. Distribuição de eventos**
- PieChart ou BarChart mostrando contagem por tipo de evento

### Padrões responsivos (seguindo convenções do projeto)
- `space-y-4 sm:space-y-6`
- `grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-4` para KPI cards
- `grid grid-cols-1 xl:grid-cols-2` para charts
- `p-4 sm:p-6` para padding
